# 请求重写与分词检索策略

## 1. 策略概述

本策略旨在解决用户请求中可能出现的错别字、语义不清等问题，通过查询重写和分词检索技术提高知识库匹配效率和准确性。

## 2. 核心技术方案

### 2.1 查询重写模块

#### 2.1.1 错别字修正
- **基于字典的错别字修正**：构建包含常见错别字的修正字典
- **优先处理多字词**：按照词长从长到短排序，避免单字错误替换
- **覆盖范围**：角色名、地区名、元素名、武器名、组织名等

#### 2.1.2 同义词扩展
- **特殊映射处理**：神明称号与实际角色名的映射（如"岩神"→"钟离"）
- **组织名称映射**：如"骑士团"→"西风骑士团"
- **属性词扩展**：如"人物"→"角色"，"属性"→"元素"

#### 2.1.3 分词优化
- **自定义分词词典**：添加原神专用词汇（角色名、地名、组织名等）
- **使用jieba分词器**：支持中文分词和自定义词典

### 2.2 检索增强策略

#### 2.2.1 知识图谱搜索
- **多维度匹配**：节点ID、标签、名称、别名、标题、属性等
- **模糊匹配算法**：基于编辑距离的模糊匹配
- **关键词匹配**：提取查询关键词进行匹配

#### 2.2.2 向量数据库检索
- **查询重写增强**：对向量查询进行重写处理
- **语义相似度匹配**：结合向量数据库的语义检索能力

#### 2.2.3 混合检索策略
- **知识图谱优先**：先尝试精确匹配和模糊匹配
- **向量检索回退**：知识图谱匹配失败时使用向量检索
- **结果排序**：根据匹配度和相关性排序

## 3. 实现细节

### 3.1 模块结构

```
modules/
├── query_rewrite.py      # 查询重写核心模块
└── langchain_integration.py  # 集成查询重写功能
```

### 3.2 核心类与方法

#### QueryRewrite类
- `correct_spelling(query)`：修正查询中的错别字
- `expand_query_with_synonyms(query)`：扩展查询的同义词
- `tokenize_query(query)`：对查询进行分词
- `extract_keywords(query)`：提取查询中的关键词
- `rewrite_query(query)`：综合处理，返回重写后的查询
- `match_node(query, node)`：匹配查询与知识图谱节点
- `search_knowledge_graph(query, knowledge_graph)`：搜索知识图谱

#### LangChainIntegration类
- `run_agent(query)`：集成查询重写功能
- `query_knowledge_base(query)`：使用重写后的查询查询向量数据库

### 3.3 字典设计

#### 错别字修正字典
包含常见的错别字映射，如：
- 角色名："钟藜"→"钟离"，"神里凌华"→"神里绫华"
- 地区名："蒙得"→"蒙德"，"璃玥"→"璃月"
- 元素名："疯"→"风"，"严"→"岩"
- 组织名："西风骑兵团"→"西风骑士团"

#### 同义词字典
包含语义相关的词汇映射，如：
- 神明："岩神"→"钟离"，"风神"→"温迪"
- 组织："骑士团"→"西风骑士团"，"七星"→"璃月七星"
- 属性："人物"→"角色"，"属性"→"元素"

## 5. 优势与特点

1. **高精度匹配**：通过错别字修正和同义词扩展，提高匹配准确率
2. **多层次检索**：结合知识图谱精确匹配和向量数据库语义检索
3. **可扩展性**：字典可以随时扩展，支持更多词汇和规则
4. **性能优化**：优先处理多字词，避免错误替换
5. **鲁棒性强**：支持各种常见的错别字和语义变化

## 6. 应用场景

- **用户查询处理**：处理用户输入中的错别字和语义不清问题
- **知识图谱检索**：提高知识图谱匹配的准确率
- **向量数据库查询**：增强向量检索的语义理解能力
- **智能问答系统**：提升问答系统的准确性和用户体验

## 7. 未来优化方向

1. **机器学习优化**：引入机器学习模型进行错别字检测和纠正
2. **动态字典更新**：根据用户查询历史动态更新修正字典
3. **上下文感知**：结合上下文信息进行更准确的查询理解
4. **多语言支持**：扩展支持多语言查询处理
5. **性能优化**：优化算法，提高处理速度
